## 1. Git 与交付规则

### 1.1 分支流程（简化 GitFlow）

1. 从 `dev` 切分支：`feature/<功能描述>-<日期>`。
2. 功能完成后合并回 `dev`。
3. 需要发布时从 `dev` 切 `release/*`，验证后合并 `main` 与 `dev`。
4. 线上紧急修复从 `main` 切 `hotfix/*`，修复后双向回合并。

### 1.2 提交与检查

- 提交规范：`feat|fix|docs|refactor|test|chore`。
- 每个 Phase 结束前至少执行：
  - 后端单测/集成测试
  - 爬虫模块测试
  - 前端构建与关键页面冒烟
- 合并前要求：文档同步更新、风险说明、回滚步骤可执行。

## 3. Phase 1 - 后端骨架与基础设施

### 目标

搭建 Spring Boot 主体工程、基础中间件接入和统一返回体系。

### 输入

- 总设计文档中的技术栈、API 契约、阿里规范清单。

### 输出

- 后端工程可启动。
- MySQL/Redis 连接可用。
- 统一响应、异常处理、日志链路（含 `traceId`）可用。

### 任务清单

1. 初始化 Spring Boot 3.2 工程与模块目录。
2. 接入 MyBatis-Plus、MySQL、Redis、Knife4j。
3. 落地 `Result`、错误码、全局异常处理。
4. 接入请求拦截与 `traceId` 生成透传。
5. 编写最小健康检查接口与测试。

### 验收标准

- 应用可本地启动。
- 健康检查接口返回统一结构且包含 `traceId`。
- 数据源与缓存连接测试通过。

### 回滚点

- 若基础依赖冲突，保留最小可启动主干，逐项恢复中间件接入。

## 4. Phase 2 - 用户与安全体系

### 目标

实现登录鉴权、权限控制、限流与黑名单能力。

### 输入

- 用户与安全相关表设计。
- 安全策略（JWT、限流、审计日志）。

### 输出

- 登录、登出、刷新 Token 接口可用。
- 核心接口具备鉴权与权限校验。
- 限流与 IP 黑名单策略生效。

### 任务清单

1. 建立用户、角色、权限、登录日志等表。
2. 实现 JWT 签发与校验、刷新机制。
3. 对核心接口加鉴权注解与权限校验。
4. 接入基于 Redis 的限流器与黑名单。
5. 增加安全审计日志（失败登录、拒绝访问、限流触发）。

### 验收标准

- 未登录访问核心接口返回 `401`。
- 无权限访问返回 `403`。
- 同源高频请求触发 `429` 且记录日志。

### 回滚点

- 若鉴权链路异常，保留只读接口白名单，逐步恢复鉴权点位。

## 5. Phase 3 - 番茄扫榜与章节抓取

### 目标

实现 V1 核心：番茄榜单、书籍详情、章节抓取与缓存。

### 输入

- 平台约束：V1 仅 `fanqie`。
- 缓存策略与数据库结构。

### 输出

- `/api/crawler/rank`、`/api/crawler/book/{id}`、`/api/crawler/chapters` 可用。
- 榜单/章节抓取结果入库与缓存。

### 任务清单

1. 初始化 FastAPI 爬虫服务与基础路由。
2. 实现番茄榜单抓取与字段清洗。
3. 实现书籍详情抓取与章节抓取（1/3/5/10章）。
4. 建立 Java-Python 服务通信与容错重试。
5. 落地缓存（榜单3天、详情7天、章节30天）。

### 验收标准

- 同参数重复请求优先命中缓存。
- 抓取失败可重试并返回可读错误。
- 书籍与章节数据可追溯到平台与时间。

### 回滚点

- 若爬虫策略失效，降级为仅返回历史缓存与友好提示。

## 6. Phase 4 - AI 分析与提示词配置

### 目标

接入 Dify + LangChain，完成三类分析与可配置提示词体系。

### 输入

- 提示词类型枚举：`deconstruct | structure | plot | theme | comprehensive`
- Dify 工作流设计与密钥策略。

### 输出

- 分析接口可用并持久化结果。
- 提示词配置可查询、修改、切换生效。
- Dify 不可用时可降级到直连模型 API。

### 任务清单

1. 实现 AI 网关层（统一请求、统一返回、统一错误处理）。
2. 接入 Dify 工作流执行与结果解析。
3. 建立 `prompt_config` 与模板管理能力。
4. 增加密钥引用 ID 或环境变量注入机制。
5. 分析结果入库并缓存。

### 验收标准

- 三类分析接口响应稳定且可追踪 `traceId`。
- 提示词修改后可在新请求生效。
- 不出现明文密钥落库或日志泄露。

### 回滚点

- 若 Dify 异常率高，切换到直连模型 API 并保留统一接口。
