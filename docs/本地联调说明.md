# 本地联调说明

## 目的

本文档记录当前仓库在 Windows 本地的联调方式，方便重新拉起前端、后端、爬虫和 Redis。

## 本机依赖

- Redis 本机目录：`D:/ProTools/redis/Redis-x64-5.0.14.1`
- JDK：`17`
- Node：`20`
- Python：`3.12`

## 启动顺序

1. 启 Redis
2. 启 crawler
3. 启 backend
4. 启 frontend

## Redis

当前本地联调用开发参数启动，避免 AOF 目录权限问题：

```powershell
& 'D:/ProTools/redis/Redis-x64-5.0.14.1/redis-server.exe' `
  --bind 127.0.0.1 `
  --port 6379 `
  --protected-mode yes `
  --appendonly no `
  --maxmemory 256mb `
  --maxmemory-policy allkeys-lru
```

说明：

- 这里关闭了 `appendonly`，只用于本地联调。
- `maxmemory 256mb` 是为了提前贴近小内存部署场景。

## Crawler

后端与 crawler 之间必须使用同一个 32 位以上的内部调用 token。

```powershell
$env:CRAWLER_INTERNAL_API_KEY = '<replace-with-32-plus-char-token>'
cd D:/Git/agent/noval/crawler
python -m uvicorn app.main:app --host 127.0.0.1 --port 5000
```

健康检查：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:5000/health
```

## Backend

当前本地 MySQL 凭据未在仓库内记录，所以联调时采用 H2 文件库兜底，不改生产配置逻辑。

H2 文件位置：

- `D:/Git/agent/noval/backend/.cache/noval-devdb-local`

启动命令：

```powershell
$env:ALIYUN_DYPNS_ENABLED = 'true'
. D:/Git/agent/noval/local.sms.env.ps1
$env:SERVER_PORT = '8080'
$env:SPRING_DATASOURCE_URL = 'jdbc:h2:file:./.cache/noval-devdb-local;MODE=MYSQL'
$env:SPRING_DATASOURCE_DRIVER_CLASS_NAME = 'org.h2.Driver'
$env:SPRING_DATASOURCE_USERNAME = 'sa'
$env:SPRING_DATASOURCE_PASSWORD = ''
$env:SPRING_SQL_INIT_MODE = 'always'
$env:SPRING_SQL_INIT_SCHEMA_LOCATIONS = 'file:./src/test/resources/sql/phase2-schema-h2.sql,file:./src/test/resources/sql/phase3-schema-h2.sql,file:./src/test/resources/sql/phase4-schema-h2.sql,file:./src/test/resources/sql/phase5-schema-h2.sql'
$env:SPRING_SQL_INIT_DATA_LOCATIONS = 'file:./src/test/resources/sql/phase2-data-h2.sql,file:./src/test/resources/sql/phase4-data-h2.sql,file:./src/test/resources/sql/phase5-data-h2.sql'
$env:SPRING_DATA_REDIS_HOST = '127.0.0.1'
$env:SPRING_DATA_REDIS_PORT = '6379'
$env:SPRING_DATA_REDIS_PASSWORD = ''
$env:CRAWLER_BASE_URL = 'http://127.0.0.1:5000'
$env:CRAWLER_INTERNAL_API_KEY = '<replace-with-the-same-token>'
$env:JWT_SECRET = 'dev-jwt-secret-change-me-please-32-characters'
$env:CONFIG_SECRET_MASTER_KEY = 'dev-config-secret-change-me-too'
$env:CLOUDFLARE_TURNSTILE_ENABLED = 'true'
$env:CLOUDFLARE_TURNSTILE_SITE_KEY = '<replace-with-turnstile-site-key>'
$env:CLOUDFLARE_TURNSTILE_SECRET_KEY = '<replace-with-turnstile-secret-key>'
$env:CLOUDFLARE_TURNSTILE_EXPECT_HOSTNAME = 'www.panch.fun'
$env:SECURITY_PASSWORD_LOGIN_PHONE_WINDOW_SECONDS = '600'
$env:SECURITY_PASSWORD_LOGIN_PHONE_MAX_FAILURES = '6'
$env:SECURITY_PASSWORD_LOGIN_IP_WINDOW_SECONDS = '600'
$env:SECURITY_PASSWORD_LOGIN_IP_MAX_FAILURES = '20'
$env:SECURITY_PASSWORD_LOGIN_PHONE_IP_WINDOW_SECONDS = '600'
$env:SECURITY_PASSWORD_LOGIN_PHONE_IP_MAX_FAILURES = '4'
$env:SECURITY_PASSWORD_LOGIN_COOLDOWN_SECONDS = '900'
$env:AI_OPENAI_COMPATIBLE_API_KEY_REF = 'DEEPSEEK_API_KEY'
$env:AI_OPENAI_COMPATIBLE_DEFAULT_MODEL = 'deepseek-chat'
$env:AI_OPENAI_COMPATIBLE_STREAMING_ENABLED = 'false'

cd D:/Git/agent/noval/backend
mvn spring-boot:run `
  '-Dspring-boot.run.useTestClasspath=true' `
  '-Dspring-boot.run.jvmArguments=-Xms256m -Xmx512m -XX:+UseSerialGC -Dfile.encoding=UTF-8'
```

补充说明：

- `CONFIG_SECRET_MASTER_KEY` 用于加密后台录入到 `system_config` / 模型注册表中的真实 key；浏览器端不会拿到明文。
- 后台系统配置新增 `auth.bootstrap-admin-phones`，默认包含 `15599316908`，命中的手机号在登录/注册/刷新时会自动补齐 `ADMIN` 角色。
- Cloudflare Turnstile 只拦截 `/api/auth/sms/send` 发码动作；普通密码登录输入不依赖它。
- 前端通过公开接口 `/api/system/auth-public-config` 拉取 Turnstile site key，后端使用 secret key 向 Cloudflare `siteverify` 校验。
- 密码登录新增三层风控：同手机号、同 IP、同手机号+IP 组合失败次数过多都会进入冷却。

健康检查：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8080/api/system/health
```

说明：

- `local.sms.env.ps1` 用于本地注入阿里云短信认证环境变量。
- 若不需要短信认证联调，可不加载该文件，并将 `ALIYUN_DYPNS_ENABLED` 设为 `false`。
