# 扫榜页与控制台轻奢改版设计

## 目标

在不改动核心业务链路的前提下，重做控制台壳层、扫榜页以及详情/抓章抽屉的视觉与交互，使界面更有美感、信息层级更清晰，同时补齐榜单完整展示和内容摘要策略。

本轮设计同时落地以下业务诉求：

- 榜单必须能展示完整内容，不再因为默认分页过小而只看到少量书。
- 每页数量改为用户可选 `5 / 10`，默认 `10`。
- 列表里的简介不展示全文，只展示约 `100` 字摘要。
- 抓章抽屉里的章节正文不直接整章铺开，只先展示摘要，点击后再看全文。
- 用户重新进入扫榜页时，默认回到自己上次看的 `channelCode + boardCode`。

## 范围

### 本轮包含

- 控制台壳层视觉重做：
  - `AppShell`
  - `AppHeader`
  - `AppSidebar`
- 扫榜页视觉与交互重做：
  - 筛选区
  - 榜单摘要区
  - 榜单列表区
  - 分页区
- 书籍详情抽屉重做
- 抓章抽屉重做
- 用户扫榜版块偏好持久化

### 本轮不包含

- 登录页重做
- 分析页、趋势页、历史页大规模改版
- 新增封面抓取或富媒体内容
- 改动 AI 分析接口协议

## 视觉方向

本轮采用“现代轻奢型内容工作台”方向：

- 背景以温暖米白与浅雾灰为主，不走纯白后台。
- 主色使用深墨绿，强调色使用克制暖金，避免廉价高饱和。
- 大标题保留内容平台气质，列表和工具区保持后台效率。
- 通过留白、卡片层级、边框与阴影强弱建立信息节奏，而不是靠大量颜色堆砌。

### 视觉原则

- 列表必须“好扫”，不能为美观牺牲效率。
- 主要交互控件保持 Element Plus 的稳定语义，但用 token、圆角、层次和布局重塑质感。
- 文案层级明显：品牌/状态/书名/作者/摘要/操作，各自有不同的视觉重量。

## 页面结构

### 1. 控制台壳层

控制台壳层拆为三层：

- 左侧导航：
  - 品牌区更简洁
  - 主导航与配置导航分区更清晰
  - 选中态更像内容工作台而不是通用后台
- 顶部栏：
  - 展示当前用户与角色
  - 保留退出按钮
  - 弱化无关说明
- 内容区：
  - 扫榜页作为主舞台
  - 内容区留白更均衡，减少“塞满整个屏幕”的压迫感

### 2. 扫榜页

扫榜页分四块：

- 顶部 Hero：
  - 当前频道 / 榜单
  - 一句简洁说明
  - 不再堆长段解释文案
- 筛选工具区：
  - 频道
  - 榜单
  - 每页显示 `5 / 10`
  - 抓章数量 `1 / 3 / 5 / 10`
  - 手动刷新榜单
- 榜单摘要区：
  - 快照 ID
  - 快照时间
  - 总书数
  - 缓存/强刷状态
- 榜单列表区：
  - 以“高信息密度内容列表卡”呈现
  - 每行展示排名、书名、作者、简介摘要、操作
  - 保留分页，不做无限滚动

## 列表展示规则

### 榜单分页

- 默认每页 `10` 本
- 用户可切换到 `5` 本
- 切换每页数量时自动回到第一页
- 继续使用后端现有分页接口，不新增新的榜单加载协议

### 简介展示

- 列表中简介统一截断到约 `100` 字
- 不在列表中展开全文
- 点击详情后，在书籍详情抽屉中展示完整简介

### 书籍详情抽屉

- 展示完整书名、作者、原始链接、完整简介
- 视觉上更像“精简档案面板”
- 摘要与正文分层，不和扫榜列表抢信息密度

## 抓章抽屉规则

### 默认态

抓章抽屉默认展示“章节摘要列表”，每章只展示：

- 章节号
- 标题
- 字数
- 正文前 `120 ~ 160` 字摘要

### 详情态

- 点击某一章后，抽屉内切换到该章详情态
- 详情态展示完整章节内容
- 支持返回章节列表

### 按钮区

抓章抽屉顶部保留：

- `进入分析页`
- `重新抓取章节`
- 重抓次数/窗口说明

## 用户偏好恢复

### 存储范围

只记住：

- `platform`
- `channelCode`
- `boardCode`

不记住：

- 页码
- 每页显示数量
- 抓章数量
- 最近打开的书

### 恢复规则

- 扫榜页初始化时，先加载榜单目录，再读取当前用户偏好
- 如果偏好里的 `channelCode + boardCode` 在当前目录中仍然存在，则直接打开该版块
- 如果该偏好已失效，则回退到当前目录里的第一个可用版块
- 用户切换频道或榜单时，立即写回偏好

## 数据与接口设计

### 后端

- 新增表：`user_rank_preference`
- 新增接口：
  - `GET /api/crawler/preference`
  - `POST /api/crawler/preference`

接口职责：

- `GET` 返回当前登录用户的扫榜版块偏好
- `POST` 保存当前登录用户的扫榜版块偏好

### 前端

- 初始化流程：
  1. 拉榜单目录
  2. 拉用户偏好
  3. 选择目标版块
  4. 刷新榜单
  5. 加载第一页
- 交互流程：
  - 切频道 / 切榜单时先保存偏好，再刷新榜单

## 文本裁剪策略

### 书籍简介

- 列表：约 `100` 字
- 详情抽屉：全文

### 章节正文

- 抓章抽屉列表：约 `120 ~ 160` 字
- 单章详情态：全文

## 测试要求

### 前端

- `RankView`
  - 偏好恢复成功时，应优先打开对应版块
  - 偏好不存在时，应回退到默认版块
  - 切频道 / 榜单时，应回写偏好
  - 每页数量切换 `5 / 10` 时，请求参数正确
- `BookDetailDrawer`
  - 详情抽屉展示完整简介
- `ChapterPreviewDrawer`
  - 默认列表展示摘要
  - 章节详情态展示全文
  - 重新抓取按钮与次数摘要正常显示

### 后端

- 用户偏好保存与读取接口
- 缺失偏好时返回合理错误或前端可降级状态
- 不影响现有榜单分页与抓章逻辑

## 风险与控制

- 风险 1：列表样式过度卡片化导致扫榜效率下降
  - 控制：保留高信息密度列表骨架，只提升层次与质感
- 风险 2：分页数量切换引发状态混乱
  - 控制：切换页大小后统一回第一页
- 风险 3：抓章抽屉里摘要/全文切换导致交互复杂
  - 控制：只做“列表态 -> 单章详情态 -> 返回列表”的单一流向

## 推荐实施顺序

1. 壳层与设计 token
2. 扫榜页布局和分页 5 / 10
3. 书籍详情抽屉与简介裁剪
4. 抓章抽屉摘要/全文分层
5. 用户版块偏好恢复与回写
