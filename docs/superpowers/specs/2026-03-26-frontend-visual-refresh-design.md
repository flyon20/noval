# 2026-03-26 前端视觉焕新设计

## 目标

在不改接口、不改页面结构主流程、不改数据模型的前提下，对前端做一轮完整的视觉焕新与移动端体验升级，让产品从当前偏厚重、偏“后台工具”的观感，升级为更年轻、更清透、可读性更强的内容型应用。

本轮重点：

- 年轻化配色与玻璃质感
- 手机端底部导航液态玻璃悬浮
- 全站亮色 / 暗色主题切换
- Markdown 结果预览的主题同步
- 趋势页图表、词云、表格与详情层的审美与可用性修正
- 抽屉和顶部栏在手机端、PC 端都更稳定可用

## 范围约束

- 不修改后端接口
- 不修改页面路由结构
- 不修改业务流程与页面主要信息架构
- 允许调整组件视觉、排版、层级、阴影、玻璃效果、明暗主题和响应式布局
- 允许为前端主题切换增加本地状态或用户配置持久化，但不增加新接口

## 当前问题

### 1. 整体视觉偏老气

- 当前主色系偏土棕、深绿，整体更像传统后台管理系统
- 卡片和导航块较厚重，缺少轻盈与透明感
- 亮色模式不够年轻，暗色模式尚未建立统一规范

### 2. 手机端导航存在存在感弱、层级不清的问题

- 底部导航虽然已固定，但悬浮感不足
- 与页面内容关系偏“压住页面”，不是“浮在内容之上”
- 缺少液态玻璃与滑动穿透感

### 3. 长文结果区域与暗色模式未统一

- Markdown 预览区仍偏“文档样式”，与整体 UI 脱节
- 暗色模式切换后需要保证结果区、卡片、代码块、列表、标题都能同步变暗
- 文字对比度必须高，不能为了视觉效果牺牲可读性

### 4. 趋势页视觉观感不稳定

- 词云不够像真正的中文词云
- 词大小、位置、留白和颜色层级还不够自然
- 饼图的标签与图面容易互相干扰
- 题材表在移动端不如卡片式结构清晰

### 5. 顶部栏和抽屉的跨端一致性不足

- 手机端顶部栏需要所有页面都可固定访问
- 各抽屉需要统一成“真正的浮层”，不能再受页面容器裁切

## 视觉方向

### 推荐风格：轻潮玻璃风

关键词：

- Glassmorphism
- Soft Gradient
- Youthful Palette
- Strong Readability
- Mobile-first polish

设计原则：

- 视觉轻盈，不厚重
- 有层次感，但不花
- 玻璃质感用于导航、按钮、卡片和浮层
- 正文阅读区优先保证清晰度
- 年轻但不过度“社交 App 化”

## 配色方案

### 亮色主题

- 背景基底：奶白、浅雾粉、浅天蓝、淡蜜桃橘渐变
- 主强调色：粉紫蓝系
- 辅助强调：蜜桃橘、草木绿、浅葡萄紫
- 危险色和提醒色保持语义，但降一点脏感和厚重感

### 暗色主题

- 背景：深灰蓝黑，而非纯黑
- 玻璃层：半透明深色卡片 + 背景模糊
- 文字：高对比浅色系，标题与正文层级清楚
- Markdown、表格、抽屉、图表全部同步暗色化

### 配色要求

- 所有正文对比度满足阅读需求
- 明暗主题都以“长文可读性”优先
- 不允许出现亮色下太灰、暗色下太发闷的问题

## 全局主题系统

### 主题变量

通过 `tokens.scss` 建立统一主题 token，覆盖：

- 背景层
- 玻璃层
- 文字层
- 强调色
- 边框
- 阴影
- 导航高度和安全区
- Markdown 预览相关变量

### 切换方式

推荐使用根节点属性切换：

- `data-theme="light"`
- `data-theme="dark"`

不改接口，通过前端本地持久化或已有用户配置保存主题偏好。

## 导航与壳层设计

### 顶部栏

- 所有页面手机端固定悬浮
- 始终能看到标题与退出入口
- 视觉上使用玻璃质感，但不遮挡正文
- 页面内容区必须自动让出顶部安全间距

### 底部导航

手机端底部导航改成液态玻璃悬浮导航：

- 浮在页面之上
- 带轻模糊和高光边缘
- 有圆角胶囊感
- 页面滚动时能感受到内容在其下方穿过
- 激活态更年轻、更亮

## 按钮与卡片

### 按钮

- 主按钮：轻玻璃高亮
- 次按钮：半透明描边玻璃
- 危险按钮：保留语义但降低“土红”感

### 卡片

- 默认卡片使用弱玻璃
- 阅读类卡片降低模糊，增强文字对比
- 图表卡片与结果卡片保持视觉一致，但结果卡片优先可读性

## Markdown 预览区

### 亮色

- 背景不再纯白，改为浅玻璃/浅纸感
- 标题、正文、引用、列表、代码块层级更清晰

### 暗色

- 深色卡片底
- 标题和正文高对比
- 代码块、引用块、分割线全部同步暗色

### 要求

- 与整体主题统一
- 不影响长文阅读
- 不为了视觉效果牺牲阅读舒适度

## 趋势页专项设计

### 词云

- 使用真正的中文词云排布
- 大词居中，小词外围散落
- 颜色更年轻、更 ins
- 避免词堆成一团
- 手机端和桌面端都保持良好留白

### 饼图

- 图内不再放密集标签
- 图外 legend / tooltip 承担详细信息
- 让图面本身更干净

### 题材表

- 桌面端：表格
- 手机端：卡片列表

### 详情层

- 趋势详情抽屉统一为真正浮层
- PC 端关闭按钮始终可见
- 内容区可滚动，不被页面裁切

## 实施顺序

### Phase 1：全局主题系统

- 重写 tokens
- 接入 light / dark theme
- 打通 App 根节点主题切换

### Phase 2：导航与壳层

- 顶部栏固定悬浮
- 底部导航液态玻璃化
- 内容区安全间距适配

### Phase 3：核心内容区

- Markdown 预览主题统一
- 结果卡和表单卡统一新风格

### Phase 4：趋势页专项优化

- 中文词云升级
- 图表 cleaner
- 表格与卡片移动端分流
- 抽屉和详情层修正

### Phase 5：全站细节收尾

- 登录 / 配置 / 历史页风格统一
- 按钮、抽屉、表格、状态色统一

## 测试与验收

### 自动化验证

- 组件测试：
  - 导航
  - 抽屉
  - 趋势词云
  - 趋势快照移动端卡片
- `npm run type-check`
- 重点页面视觉行为回归测试

### 人工验收

- 手机端底部导航是否具备悬浮玻璃感
- 手机端顶部栏是否始终可见
- 暗色模式下 Markdown 是否同步变暗
- 长文是否仍然易读
- 词云是否不再挤成一坨
- 手机端趋势表是否不再混乱

## 风险控制

- 不一次性改动业务逻辑
- 保持现有组件边界，优先做主题和样式替换
- 先修壳层和通用 token，再修局部组件
- 每一阶段完成后做一次类型检查和局部测试

## 本轮不做

- 不改接口
- 不改页面数据结构
- 不新增复杂的设计系统依赖
- 不重构业务路由
